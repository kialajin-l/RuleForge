## 📝 活动正文

## 问题背景（痛点）

作为开发者，你是否经常遇到这些问题：

- **重复造轮子**：同样的错误反复出现，每次都要重新搜索解决方案
- **最佳实践流失**：团队中的经验难以沉淀，新人总是踩同样的坑
- **规范执行困难**：代码审查耗时耗力，规范文档形同虚设
- **知识孤岛**：个人经验无法有效分享给社区

据统计，开发者平均 **30% 的时间** 花在解决已经解决过的问题上。

## 解决方案（创新点）

RuleForge 是一个智能规则提取引擎，能够：

- ✅ **自动提取**：从开发会话中识别最佳实践，无需手动整理
- ✅ **标准化格式**：REP v0.1 协议，工具无关，跨平台兼容
- ✅ **智能评分**：置信度算法评估规则质量，确保可靠性
- ✅ **一键共享**：贡献至开源社区，形成规范生态
- ✅ **隐私保护**：自动脱敏敏感信息，安全无忧

## 技术架构（亮点）

### 核心技术栈
- **TypeScript + Node.js** 全栈开发，类型安全
- **Zod Schema** 验证确保规则质量
- **多阶段提取 pipeline**：解析 → 聚类 → 格式化
- **智能置信度评分**：`occurrence_count * success_rate * applicability`

### 架构设计
```
┌─────────────────────────────────────────────┐
│              RuleForge 架构                  │
├─────────────────────────────────────────────┤
│  @ruleforge/cli (命令行工具)                 │
│  ├── init: 项目初始化                        │
│  ├── extract: 规则提取                       │
│  ├── validate: 规则验证                      │
│  └── list/show/delete: 规则管理              │
├─────────────────────────────────────────────┤
│  @ruleforge/core (核心引擎)                  │
│  ├── LogParser: 日志解析                     │
│  ├── PatternCluster: 模式聚类                │
│  ├── RuleExtractor: 规则提取                 │
│  ├── RuleValidator: 规则验证                 │
│  └── YamlFormatter: YAML 格式化              │
├─────────────────────────────────────────────┤
│  @ruleforge/adapter-* (插件适配)             │
│  ├── VSCode Extension                        │
│  ├── Trae Plugin                             │
│  └── GitHub Action                           │
└─────────────────────────────────────────────┘
```

### 关键算法
```typescript
// 置信度评分算法
confidence = occurrence_count * success_rate * applicability

// 模式聚类
similarity = levenshtein_distance(code_a, code_b) / max_length
```

### 快速开始
```bash
# 安装 CLI
npm install -g @ruleforge/cli

# 初始化项目
ruleforge init --template vue

# 提取规则
ruleforge extract --log ./session.jsonl

# 验证规则
ruleforge validate ./rule.yaml
```
